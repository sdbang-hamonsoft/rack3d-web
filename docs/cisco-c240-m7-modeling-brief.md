# Cisco UCS C240 M7 — Blender MCP 모델링 사양서

목적: `public/models/cisco-ucs-c240-m7.glb`를 앞면/뒷면 위주로 더 정교하게 재모델링.
용도: Three.js(@react-three/fiber) 웹 뷰어. → "형상 정확 + 가벼운 폴리곤 + 단순 재질" 우선.
드라이브 구성: **SFF 24×2.5" (기본)**.

## 0. 시작 전 (모델링 세션에서)
- Blender MCP 연결 확인: 애드온 서버 포트 9876, `claude mcp list`에 `blender ✔`.
- **스케일/방향 기준 맞추기**: 기존 `public/models/dell-poweredge-r760.glb`를 Blender로 import해서
  단위·축·앞면 방향(어느 축이 앞인지)·랙 안에서의 원점 위치를 확인하고 C240도 동일 규격으로 맞춘다.
- 앱 스케일: `App.tsx`의 `UNIT_HEIGHT = 0.04445` (1U = 44.45mm). 2U 서버 높이 = 0.0889m.

## 1. 섀시 (Chassis) — 2U
- 치수(실측 근사): 높이 **88.9mm(2U)** × 폭 **448mm(19" 랙)** × 깊이 **760mm**.
- 앞→뒤 = 깊이 방향. 살짝 라운드된 엣지(bevel 1~2mm) 정도만.
- 재질: 다크 그레이 메탈 (metalness 0.6, roughness 0.5).

## 2. 앞면 (Front) — SFF 24×2.5"
- **랙 이어 2개** (좌/우 돌출):
  - 우측 이어: 전원 버튼, 상태 LED 스택(전원/상태/네트워크 액티비티), Unit ID 버튼.
  - 좌측 이어: 풀아웃 에셋 태그(얇은 슬롯) + KVM/진단 포트 흔적.
- **드라이브 베이 24개**: 세로로 세운 2.5" 캐디를 가로로 24열 배열(보통 24열 1행).
  - 각 캐디: 얇은 직사각 베젤 + 좌측 릴리즈 래치(작은 돌기) + 2개 LED 점(활동/상태).
  - 캐디 사이 1mm 갭.
- **Cisco 로고 바**: 우측 이어 근처 상단에 얇은 실크스크린 영역(단순 평면 + 밝은 재질/텍스트 텍스처).
- 통풍 홀 패턴은 노멀맵 대체 가능(폴리곤 절약). 실루엣상 필요한 것만 지오메트리로.

## 3. 뒷면 (Rear)
- **PSU 2개**: 우측에 상하로 2개(또는 좌우), 각 팬 그릴(원형 함몰) + 손잡이 + AC 인렛.
- **PCIe 라이저 슬롯**: 상단에 가로로 배열된 확장 슬롯 브래킷(빈 슬롯 커버 3~6개).
- **mLOM / 리어 I/O 패널**:
  - VGA(사다리꼴), 시리얼 RJ45, USB 3.0 ×2(직사각), 1GbE 관리 포트, 듀얼 LOM 포트.
  - 각 포트는 실제 커넥터 실루엣 정도로 단순화.
- 재질: 앞면과 동일 계열, 포트 내부는 어둡게.

## 4. 폴리곤/최적화 지침
- 웹뷰어 타깃: 전체 20~60k tris 목표. 미러 모디파이어로 좌우 대칭 활용.
- 반복 요소(드라이브 캐디, PSU)는 array/linked-duplicate로 생성.
- 앞/뒤 외 측면·내부는 최소화(안 보임).

## 5. 익스포트
- glTF 2.0 Binary(.glb), Y-up, +Z front(기존 모델 방향에 맞춤).
- 파일: `public/models/cisco-ucs-c240-m7.glb` 로 덮어쓰기 전
  기존 파일을 `cisco-ucs-c240-m7.prev.glb`로 백업.
- 익스포트 후 `npm run dev`로 랙 안에서 크기·방향 육안 확인.

## 참고 실측치 (근사, 검증 후 사용)
- 2U = 88.9mm, 19" 랙 폭 섀시 ≈ 448mm, 깊이 ≈ 760mm.
- 2.5" 드라이브 캐디 폭 ≈ 17~18mm → 24개 + 이어로 폭이 대략 맞아떨어짐.
